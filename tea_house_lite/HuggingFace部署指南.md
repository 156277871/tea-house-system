# 🚀 Hugging Face Spaces 部署指南

**无需 GitHub 账号，无需编程基础，5分钟免费部署茶楼管理系统！**

---

## 📋 准备工作

您需要准备：
- ✅ 一个邮箱（QQ、163、Gmail 都可以）
- ✅ 一个手机号（用于注册验证）
- ✅ 3个文件（已为您准备好）

---

## 📝 第一步：注册 Hugging Face 账号

### 1.1 打开注册页面
访问：https://huggingface.co/join

### 1.2 填写注册信息
- **用户名**：输入一个容易记住的名字（例如：zhangsan123）
- **邮箱**：输入您的邮箱地址
- **密码**：设置一个安全的密码

### 1.3 验证邮箱
- 登录您的邮箱
- 找到来自 Hugging Face 的验证邮件（可能在垃圾箱里）
- 点击验证链接

### 1.4 验证手机号（可选，但建议完成）
- 进入个人设置页面
- 添加手机号并完成验证

---

## 🆕 第二步：创建新的 Space

### 2.1 创建 Space
访问：https://huggingface.co/new-space

### 2.2 填写 Space 信息

**Owner**（所有者）：
- 选择您的用户名

**Space name**（Space 名称）：
- 输入：tea-house-system
- 或者输入您喜欢的英文名称（只能用字母、数字、短横线）

**License**（许可证）：
- 选择：MIT

**Select the SDK**（选择 SDK）：
- **重要！** 选择：Streamlit

**Make your Space public or private**（公开或私有）：
- 选择：Public（公开，免费）

### 2.3 创建 Space
点击页面底部的 **"Create Space"** 按钮

---

## 📤 第三步：上传文件

### 3.1 下载文件包
从对象存储下载以下3个文件：
- `app.py` - 主程序文件
- `requirements.txt` - 依赖文件
- `README.md` - 说明文件

### 3.2 打开您创建的 Space
页面会自动跳转到您的 Space，或者访问：
```
https://huggingface.co/spaces/您的用户名/tea-house-system
```

### 3.3 上传文件（方法一：网页上传）
1. 点击页面右侧的 **"Files and versions"** 标签
2. 点击 **"Add file"** → **"Upload files"**
3. 依次上传3个文件：
   - 点击 **"Browse files"**
   - 选择电脑上的 `app.py` 文件
   - 点击 **"Commit changes to main"**
4. 重复步骤3，上传 `requirements.txt` 和 `README.md`

### 3.4 上传文件（方法二：批量上传）
1. 点击页面右侧的 **"Files and versions"** 标签
2. 点击 **"Add file"** → **"Upload files"**
3. **一次性选择所有3个文件**（按住Ctrl键多选）
4. 点击 **"Commit changes to main"**

---

## ⏳ 第四步：等待部署完成

### 4.1 自动构建
- 文件上传后，Hugging Face 会自动开始构建
- 这需要 **2-5分钟**，请耐心等待

### 4.2 查看构建状态
- 在 **"App"** 标签页可以看到构建日志
- 状态显示 **"Running"** 即表示部署成功

### 4.3 可能遇到的问题
**如果构建失败：**
- 检查文件名是否正确（app.py、requirements.txt、README.md）
- 确保上传了所有3个文件
- 查看 **"Logs"** 标签中的错误信息
- 点击页面右上角的 **"Settings"** → **"Factory reset"** 重置后重新上传

---

## 🎉 第五步：开始使用

### 5.1 访问您的系统
部署成功后，访问：
```
https://您的用户名-tea-house-system.hf.space
```

### 5.2 初始化数据
按照以下顺序创建基础数据：

**步骤1：创建门店**
- 点击左侧菜单 **"🏪 门店管理"**
- 切换到 **"新增门店"** 标签
- 填写：
  - 门店名称：例如"北京朝阳店"
  - 门店编码：例如"BJ001"
  - 地址、电话（选填）
- 点击 **"创建"**

**步骤2：创建商品**
- 点击左侧菜单 **"🛍️ 商品管理"**
- 切换到 **"新增商品"** 标签
- 填写：
  - 商品名称：例如"龙井茶"
  - 商品编码：例如"P001"
  - 分类：选择（茶叶/茶具/点心/饮品）
  - 单价：例如 128
  - 单位：例如 "盒"
- 点击 **"创建"**

**步骤3：创建员工**
- 点击左侧菜单 **"👥 员工管理"**
- 切换到 **"新增员工"** 标签
- 填写：
  - 姓名：例如"张三"
  - 电话：例如"13800138000"
  - 职位：选择（店长/店员/收银员）
  - 所属门店：选择刚才创建的门店
- 点击 **"创建"**

**步骤4：创建会员**
- 点击左侧菜单 **"💎 会员管理"**
- 切换到 **"新增会员"** 标签
- 填写：
  - 姓名：例如"李四"
  - 电话：例如"13900139000"
- 点击 **"创建"**

**步骤5：库存入库**
- 点击左侧菜单 **"📦 库存管理"**
- 切换到 **"库存入库"** 标签
- 选择门店和商品
- 输入入库数量
- 点击 **"入库"**

**步骤6：创建订单**
- 点击左侧菜单 **"📝 订单管理"**
- 切换到 **"创建订单"** 标签
- 选择门店、支付方式、商品和数量
- 点击 **"创建"**

## ⚠️ 重要提示

### 关于数据保存
- 本系统使用 SQLite 数据库
- **每次 Space 重启后，数据会重置**
- 重启情况包括：
  - 24小时未访问
  - Hugging Face 系统维护
  - 手动重启

### 如何避免数据丢失？
1. **定期查看控制台**，保持 Space 活跃
2. **记录重要数据**（如商品信息、会员信息）
3. 如需永久保存数据，考虑升级到付费版或自建服务器

### 关于访问速度
- 首次访问可能需要加载 10-30 秒
- 免费版有一定流量限制
- 如需更高性能，可考虑升级到付费版

## 🆘 常见问题

### Q1: 注册 Hugging Face 时收不到验证邮件？
**A:** 检查垃圾箱，或使用 Gmail/Outlook 邮箱

### Q2: 上传文件后一直显示 "Building"？
**A:** 正常现象，等待2-5分钟即可

### Q3: 访问页面时显示 404 错误？
**A:** 等待部署完成后再访问，检查URL是否正确

### Q4: 数据隔天就消失了？
**A:** 这是免费版的限制，见"重要提示"部分

### Q5: 可以让多个员工同时使用吗？
**A:** 可以，分享URL给员工即可

### Q6: 可以手机上使用吗？
**A:** 可以，在手机浏览器打开URL即可
